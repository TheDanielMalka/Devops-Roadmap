1. השלמות ב-Jira (החפירה שביקשת)
כרגע הגדרנו את ה"צינור" (חיבור לגיט, העברה אוטומטית ל-Done). כדי להבין את ה-Workflow של המפתחים לעומק, חסר לנו:

Backlog vs Board: איך מנהלים ערימת משימות לעומת מה שקורה "עכשיו".

Sprints: איך מחלקים עבודה לזמן קצוב (שבועיים בדרך כלל).

Smart Commits מתקדמים: לא רק #comment או #done. נלמד איך לרשום זמן עבודה (#time 2h 30m) ישירות מה-Git, מה שחוסך למפתחים המון זמן בדיווחים.

Components & Labels: איך מחלקים פרויקט גדול (למשל: Frontend, Backend, Database) בתוך ה-Jira כדי שיהיה סדר.

2. Git Workflow (היעד הבא שלנו)
זה הלב של העבודה בצוות. נלמד את ההבדלים בין:

Git Flow: העבודה ה"קלאסית" עם ענפי Develop, Feature, ו-Master.

GitHub Flow: המודל המודרני והמהיר יותר (שבו הכל רץ סביב ה-Main).

Trunk Based Development: מה שחברות ענק (כמו גוגל ופייסבוק) עושות.

תרגול: נבצע "Merge Conflict" בכוונה ונלמד איך לפתור אותו בלי להרוס את הקוד.

3. Build & Packaging Tools (מה שלא נגענו)
כאן נבין איך קוד הופך ל"מוצר".

נלמד מה זה Artifact.

נכיר כלי בנייה לפי שפה: למשל Maven/Gradle (ל-Java), NPM (ל-JS/Node), או PyPI (ל-Python).

נבין מה זה Docker (בקטנה ממש) בהקשר של אריזה.

4. Automated Testing (הבנת המושגים)
אתה לא צריך להיות בודק תוכנה, אבל כאיש DevOps אתה חייב לדעת איפה הבדיקה נכנסת ב-Pipeline:

Unit Tests: בדיקת הפונקציה הקטנה.

Integration Tests: בדיקה שהכל מתחבר.

CI (Continuous Integration): איך ה-GitHub Action שכתבנו יריץ את הבדיקות האלו אוטומטית לפני שהוא נותן למזג את הקוד.

איך נתרגל את זה בשבועיים?
אני מציע לעבוד בשיטה של "יום תיאוריה - יום יישום":

מחר: נחפור ב-Jira על ניהול ספרינטים ו-Backlog (נבנה "ספרינט" לשבועיים הקרובים שלנו בתוך ה-Jira שלך).

מחרתיים: נתחיל את ה-Git Workflow הראשון שלנו.

לפי התוכנית הזו, תרצה שנתחיל מחר בלבנות "Sprint" ב-Jira שיכיל את כל המשימות של הלימודים שלנו לשבועיים הקרובים? (ככה גם תתרגל Jira וגם יהיה לנו סדר).
